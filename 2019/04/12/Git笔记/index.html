Git笔记

1.git是分布式版本控制工具
2.git分为工作区->暂存区->本地仓库
3.git基本操作

  • 1.初始化本地仓库 git init .git文件
  • 2.设置签名
    用户级别
    git config user.name XXX
    git config user.email xxx
    系统级别
    git config -global user.name xxx
    git config -global user.email xxx
  • 3.加入暂存区 git add [fileName]
  • 4.提交到本地库 git commit -m “xxx” [fileName]
  • 5.查看仓库状态 git status
  • 6.回退上一级状态 git rm –cached
  • 7.查看历史版本 git log /git relog git /git log –oneline(只显示过去)

  • 8.版本回退
    索引操作
    git reset –hard index
    使用^符号后退
    git reset –hard HEAD^
    使用~符号后退
    git reset –hard HEAD~n

  • 9.比较文件 git diff fileName;git diff HEAD;git diff HEAD fileName

4.HEAD指针移动方式

  • 1.soft本地库移动
  • 2.mixed本地库移动，暂存区移动
  • 3.hard本地库移动，暂存区移动，工作区移动

5.删除文件并找回

  • 1.git init
  • 2.vim fileName
  • 3.git add fileName
  • 4.git commit fileName
  • 5.vim fileName
  • 6.rm fileName
  • 7.git reset –hard index
    or
  • 1.git init
  • 2.vim fileName
  • 3.git add fileName
  • 4.git commit fileName
  • 5.rm fileName
  • 6.git reset –hard HEAD

6.分支操作

  • 1.查看分支 git branch -v
  • 2.创建分支 git add branch branchName
  • 3.切换分支 git checkout branchName
  • 4.合并分支 git merge branchName->B A 合并 B
  • 5.master branch：主分支
  • 6.develop branch：开发分支
  • 7.feature branch：功能分支
  • 8.release branch：预发布分支
  • 9.hotfix branch：bug修复分支

7.分支冲突

  • 1.原因:A分支修改文件提交 B分支修改同一个文件提交 仓库不知道以哪个为标准
  • 2.解决办法:
    编辑文件删除特殊符号
    git add fileName
    git commit -m “” 不带文件名

8.Git原理

  • 1.Hash 算法:SHA-1算法
  • 2.版本控制:链表
  • 3.分支管理:HEAD指针指向分支

9.协同开发

  • 1.本地库上传到远程库
    git remote -v 查看远程库
    git remote add XXX https://wwww.xxxx.git 命名远程库
    git push xxx master(branch) 上传到远程库
  • 2.远程库克隆到本地库
    git clone https://www.xxxx.git
    效果:完整将远程库下载到本地库且初始化了本地库
  • 3.加入成员从本地库上传到远程库
    1.setting->Collaborators->email
    2.复制邀请链接发送给邀请人->accept
    3.git push xxx xxx
  • 4.从本地库拉取项目
    git pull xxx xxx / git fetch xxx xxx pull=fetch+merg
